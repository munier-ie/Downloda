# Security Policy

We take the security of Downloda seriously. If you believe you have found a security vulnerability, please report it to us privately using the procedure outlined below.

---

## Supported Versions

Only the latest active stable release receives security updates and patches. We recommend all users keep their application updated to the latest available version.

| Version | Supported |
| :--- | :---: |
| >= 1.0.0 | Yes |
| < 1.0.0 | No |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a vulnerability, please report it by sending an email to the project maintainers at **security@downloda.app** (or the email specified in the repository profile). 

In your report, please include:
* A detailed description of the vulnerability.
* The potential impact or exploit scenarios.
* Steps to reproduce the issue (including proof-of-concept scripts or steps).
* The version of Downloda, Android OS, and device model tested.

We will acknowledge receipt of your report within 48 hours and work with you to coordinate a security patch and public disclosure.

---

## Scope

This security policy applies only to the source code and official release builds distributed through the official Downloda GitHub repository. Third-party forks or modifications are not covered under this policy.
